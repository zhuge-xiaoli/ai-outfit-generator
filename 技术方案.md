## 二、技术栈详情

### 2.1 前端

| 技术 | 用途 | 理由 |
|------|------|------|
| **Vue 3** | UI框架 | 响应式开发，CDN引入免构建 |
| **Element Plus** | UI组件库 | 现代化组件，上传/按钮等 |
| **Axios** | HTTP请求 | 与后端通信 |

### 2.2 后端

| 技术 | 用途 | 理由 |
|------|------|------|
| **FastAPI** | Web框架 | 异步高性能，Python生态 |
| **Uvicorn** | ASGI服务器 | 运行FastAPI |
| **Pydantic** | 数据验证 | 请求/响应校验 |
| **Python-dotenv** | 环境变量 | 管理API密钥 |

### 2.3 AI服务（国内）

| 服务 | 用途 | 说明 |
|------|------|------|
| **DeepSeek-VL** | 视觉理解+文案生成 | 推荐，支持图像理解 |
| **MiniMax M2** | 视觉理解+文案生成 | 支持多模态 |
| **讯飞星火** | 视觉理解+文案生成 | 国内可用 |

## 三、核心模块设计

### 3.1 图片上传模块

```python

const file = input.files[0]
const base64 = await fileToBase64(file)

# 后端：接收Base64
@app.post("/api/generate")

    image_data = request.image  # Base64编码
```

### 3.2 LLM分析模块

**Prompt模板（图片分析）**：
```
请分析这张图片中的服装：
1. 服装类型（外套、上衣、裤子、鞋子等）
2. 颜色（具体描述）
3. 款式特征（宽松、修身、细节设计）
4. 面料质感（看起来像什么材质）
5. 适用场景

请用简洁的结构化格式输出。
```

### 3.3 提示词生成模块

**提示词模板结构**：
```
{模特描述} + {动作描述} + {服装描述} + {场景描述} + {光线摄影参数}
```

**预设动作库**：
| 动作 | 描述 |
|------|------|
| 包带互动 | 右手握胸前斜挎包带，左手插兜 |
| 慵懒半靠墙 | 身体微靠墙，一条腿前伸 |
| 动态大步流星 | 迎面迈步走，手持咖啡 |
| 单手推车 | 超市场景，单手推购物车 |
| 比耶抓拍 | 抬手比耶手势 |

### 3.4 文案生成模块

**小红书文案风格**：
- 标题：带入感 + 情绪词 + 悬念
- 正文：第一人称叙事 + emoji + 互动引导
- 标签：#穿搭 #男友穿搭 等

## 四、API设计

### 4.1 接口：生成提示词和文案

**请求**：
```json

{
  "image": "data:image/jpeg;base64,...",
  "action_type": "包带互动",
  "scene_description": "可选的场景描述"
```

## 七、LLM调用示例

### 7.1 DeepSeek-VL 调用

```python
import openai

client = openai.OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com/v1"
)

response = client.chat.completions.create(
    model="deepseek-chat",  # DeepSeek-VL 需要用 vision 模型
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image_url",
                    "image_url": {"url": f"data:image/jpeg;base64,{image_base64}"}
                },
                {
                    "type": "text",
                    "text": "请分析这张图片中的服装特征..."
                }
            ]
        }
    ],
    max_tokens=2000
)
```

## 九、注意事项

1. **API成本**：图片转Base64后发送给LLM会有token费用，注意控制图片大小
2. **错误处理**：网络超时、API限流等需要友好提示
3. **安全性**：前端不存储用户图片，后端处理完即可删除
